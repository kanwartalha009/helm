## 5. Anything alarming

1. The `/dashboard` brand table is mock data (`DashboardPage.tsx:22`); the real aggregation exists but is unused. A demo of the headline screen shows fabricated numbers.
2. Zero automated tests (`api/tests` absent) on a system doing money aggregation with a recent sync rewrite. Any regression ships silently.
3. DB password `CQxTjp6njA` sits in plaintext in `.env.production.recovered`, committed to the repo tree. Rotate and purge from git history.
4. `MAIL_MAILER=log` — the first manager/brand-user invitation writes to a log file and never sends. Phase 1.5 onboarding is dead on arrival until fixed.
5. `SENTRY_LARAVEL_DSN` empty — combined with the permanent-connection policy (failed sync stays `active`), failures are observable only via sync_logs, not alerts.
6. FX is Schrödinger's feature: column nullable, jobs hardcode `1.0`, fetch unscheduled, but the USD toggle is a Phase 1 acceptance criterion (docs/12:19). The toggle will show native==USD silently. Decide before sign-off.
7. Trigger/cron connection-filter mismatch (`!= 'paused'` vs `= 'active'`): an `errored` connection syncs from manual buttons but is skipped by every cron. Inconsistent and surprising.
8. Master MFA is opt-in, not enforced (`AuthController.php:52`), contradicting docs/08 + docs/12:31. A master_admin can run forever without a second factor.
